# Helm Template Output

This directory contains the rendered output of `helm template` for the Loculus helm chart with different values configurations.

These files are automatically generated by the CI workflow (`.github/workflows/helm-template-output.yml`) and should not be manually edited.

## Directory Structure

The output is organized by configuration and split into individual files based on their source templates:

```
rendered/
├── default/              # Output using default values.yaml
│   ├── loculus-backend.yaml
│   ├── loculus-website.yaml
│   ├── secrets.yaml
│   └── ...
├── e2e-dev/              # Output using values.yaml + values_e2e_and_dev.yaml
│   ├── loculus-backend.yaml
│   └── ...
└── preview-server/       # Output using values.yaml + values_preview_server.yaml
    ├── loculus-backend.yaml
    └── ...
```

Each file corresponds to a template from `kubernetes/loculus/templates/` and contains all the Kubernetes resources rendered from that template.

## Purpose

These files serve as documentation to help reviewers understand how changes to helm templates and values affect the final rendered Kubernetes manifests. By splitting the output into separate files:

- It's easier to review changes to specific components
- Diffs are more focused and readable
- You can quickly see which templates are affected by your changes

## Updating

When making changes to helm templates or values files, the CI will detect changes to these rendered outputs. If changes are detected:

1. Review the diff to ensure the changes are intentional
2. Add the `update_helm_output` label to your PR
3. The CI will automatically commit the updated rendered outputs

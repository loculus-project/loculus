# Loculus March 2026 Update

*By the Loculus Team - 17 March 2026*

Spring is on the horizon, and the Loculus team has been busy shipping improvements across search, preprocessing, and deployment flexibility. This month's highlights include a new segment filter for multi-segmented organisms, SeqSet search from the search bar, a percentage-based completeness filter, and a powerful new NCBI taxonomy database for preprocessing. Read on for a roundup of all the notable changes!

## Search and Filtering

- **Required Segments filter for multi-segmented viruses:** A new "Required Segments" section with checkboxes now appears at the top of the search filters panel for multi-segmented organisms. Previously, users had to manually set length filters greater than zero for each segment - now, simply check which segments you need. This filter only appears for multi-segmented organisms, keeping the UI clean for single-segment pathogens. ([#6057](https://github.com/loculus-project/loculus/pull/6057))

- **SeqSet search in the search bar:** The main search bar now supports SeqSet lookups. When you enter an accession with a SeqSet prefix, the search bar automatically routes to a SeqSet lookup, falling back to a regular accession search otherwise. This makes finding SeqSets faster and more intuitive. ([#6126](https://github.com/loculus-project/loculus/pull/6126))

- **Completeness filter displayed as percentage:** The completeness filter now displays as a familiar percentage (0-100%) rather than a raw decimal (0-1). Active filter badges show values like "95%" instead of "0.95", and the input label includes "(%)". Values are automatically converted before being sent to the API. ([#6049](https://github.com/loculus-project/loculus/pull/6049))

- **Single-choice for numeric filter fields:** Numeric (integer and float) filter fields now only allow single-value selection, since the backend doesn't support multi-value queries for these types. This prevents confusing error messages that users previously encountered. ([#6125](https://github.com/loculus-project/loculus/pull/6125))

## Display and UI Improvements

- **Display names for file categories:** The upload area now shows human-readable display names for file categories (e.g. for raw reads), rather than raw internal identifiers. This makes the submission interface clearer and more approachable. ([#6131](https://github.com/loculus-project/loculus/pull/6131))

- **Cleaner download UI for single-segmented organisms:** For organisms with only one segment, the download dialog no longer shows the internal segment name (e.g. "main"), which was confusing since segment names aren't surfaced elsewhere for these organisms. ([#6107](https://github.com/loculus-project/loculus/pull/6107))

- **Configurable ID field descriptions:** The `id` and `fastaIds` input fields are now generated by the Helm template and configurable per organism. Default descriptions are context-aware for single-segment vs. multi-segment organisms, and backtick-quoted text in field descriptions now renders as styled inline code. ([#6077](https://github.com/loculus-project/loculus/pull/6077))

- **Blockquote styling in Markdown pages:** Markdown pages rendered with MdLayout now properly style blockquotes with a left border, muted gray text, and italic formatting - previously they were visually indistinguishable from regular text. ([#6113](https://github.com/loculus-project/loculus/pull/6113))

## Preprocessing

- **NCBI taxonomy database:** A new module downloads NCBI's taxonomy data and builds a local SQLite database for use during preprocessing. This enables validation of scientific host names, mapping to common names, and hierarchical taxonomy filtering (e.g. searching for "mosquito" includes all mosquito species). A GitHub Action runs monthly to keep the database updated. ([#6018](https://github.com/loculus-project/loculus/pull/6018))

- **Standardized display names:** A new structured `build_display_name` function generates display names from `specimenCollectorSampleId` (or `submissionId`) and metadata in a consistent way, with optional ID extraction via regex from recognized formats. ([#6066](https://github.com/loculus-project/loculus/pull/6066))

- **Configurable string concatenation:** The `build_display_name` function now supports configurable string concatenation with specified prefixes, field ordering, identifier extraction via regex patterns, and type specification for each component. ([#6110](https://github.com/loculus-project/loculus/pull/6110))

## Deployment and Configuration

- **Configurable contact message for issues:** A new `gitHubIssuesUrl` and `issuesEmail` configuration in `values.yaml` lets deployers customize the contact/support message shown to users when issues occur. A reusable `ErrorContactMessage` component is now available throughout the frontend. Note: this is a breaking change that requires updating deployment configuration. ([#6103](https://github.com/loculus-project/loculus/pull/6103))

## Deposition (ENA Submission)

- **Deduplicated author affiliations:** Author affiliations are now deduplicated before submission to ENA, fixing an issue where affiliations could appear multiple times in submitted data. ([#6073](https://github.com/loculus-project/loculus/pull/6073))

- **Timestamp always included in assembly names:** Assembly names submitted to ENA now always include a timestamp, preventing a state where the submission pod could not retry failed submissions and required manual intervention. ([#6053](https://github.com/loculus-project/loculus/pull/6053))

## DevOps

- **YAML linting action:** A new `yamlfmt-lint` GitHub Action enforces consistent YAML formatting across the codebase using Google's `yamlfmt`. All existing YAML files have been reformatted as part of this change. ([#6075](https://github.com/loculus-project/loculus/pull/6075))
